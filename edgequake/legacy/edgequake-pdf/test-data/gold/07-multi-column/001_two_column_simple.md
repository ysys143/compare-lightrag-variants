# Two Column Text 1

This is the start of a two-column document. The text flows across the page in two columns to test how well the converter handles multi-column layouts.

Column one contains the first paragraph with content that should be extracted in the correct reading order, from top to bottom of the left column first, then continuing to the right column.

Testing how paragraph breaks are preserved when text is split across columns.

The converter needs to detect the column boundaries and reassemble the text in the proper reading order rather than extracting left-to-right line by line.

This is particularly important for documents like newspapers, magazines, and academic papers that use multi-column layouts.
